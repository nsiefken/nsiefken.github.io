Classes v. Modules

4-22-15

I wasn't (and still am not) very familiar with modules, which is the reason I wanted to Blog about them today. I've been pretty curious about classes v. modules myself, so I when I saw that on the challenge I knew they must be pretty similar. And it seems they are. Classes and modules are both used to group a bunch of code together, code that you will often want to type when building a giant program. They are tools to help you group specific lines of code together and easily use them over and over again.

My understanding is modules can be used as a separate storage compartment when you need it. Classes are used to create instances but modules are not. Modules can collect methods and constants (and even classes!) but modules aren't used for instances.

Modules can be used to store pieces of code that can then be used across multiple classes, let's say. If you look up modules (like I had to), you will often read that this helps prevent name clashes. If two methods or even two classes got named the same thing, modules can store them differently so your program knows which one to call on at the right time. Just think of modules as another layer of which to group and store your code.
