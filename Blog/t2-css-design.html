CSS Positioning

The Difference Between Absolute, Relative, and Fixed Positioning

3-27-15

Positioning elements is one of the most important features to CSS, but can also be extremely confusing. In fact, I wrote this blog post b/c I was confused about positioning myself. Here I will go through the differences between Absolute, Relative, and Fixed positioning. There's a fourth type called Static, but that is the default position for any element, and is rarely ever used in CSS.

The first type of positioning is Absolute. Absolute positioning allows you to place an object wherever you want on the screen. The coordinates you type in are based on the screen. So if you type in

Selector {

position: absolute;

top: 0px;

left: 0px;

}

the element will be place in the top left corner of your screen. Absolute positioning can be great and easy, but it also takes elements out from the flow of the page, meaning it's difficult to use with other objects. This is where Relative Positioning comes in

Relative Positioning places an object relative to itself and where it fits normally on the page. If you type in the same code as above but use relative instead of absolute, the object won't move at all because you told it to move from its top 0px and from its left 0px.

Fixed Positioning essentially glues an object to the page so when you scroll the object an still be seen. It is great if you want a navagational bar to always be seen, but it can be dangerous because you don't want a fixed object to cover up other objects as you scroll down the page. Each type of position has its own pros and cons and figuring out which position is the best all depends on what type of design you want for your site.
